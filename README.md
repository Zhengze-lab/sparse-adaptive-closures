# Sparse Adaptive Closures: Experimental Code

This repository contains the experimental code and reproducibility artifacts for sparse adaptive closures in regime-aware gray-box ODE models.

The core idea implemented here is to keep a locally valid gray-box ODE as the trusted backbone and learn sparse state/input/condition-dependent functions only for screened physical coefficient slots.

This repository intentionally contains **experiment code only**: executable scripts, compact derived data, provenance records, frozen metrics, and regenerable experiment figures.

## Installation

Use Python 3.11 or newer. The local development environment used Python 3.13 and the versions pinned in `requirements.txt`.

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

Conda users can instead run:

```bash
conda env create -f environment.yml
conda activate sparse-adaptive-closures
```

Some public real-system scripts require third-party datasets that are downloaded or cached by their original providers. The synthetic E1--E5 experiments do not require external raw data.

## Quick Reproduction

Run from the repository root.

Smoke test:

```bash
python scripts/smoke_test.py
```

Core synthetic cases:

```bash
python scripts/run_e3_local_linearization_pendulum.py
python scripts/run_e4_velocity_drag.py
python scripts/run_e5_monod_rate_slot.py
```

Slot-screening protocol:

```bash
python scripts/run_slot_screening.py
```

Additional ablations:

```bash
python scripts/run_ablation_suite.py
```

Release-package QA:

```bash
python scripts/check_release_package.py
```

## Main Experiment Families

- `E1/E2`: mechanism checks on Van der Pol and Duffing-style coefficient slots.
- `E3`: local-linear pendulum upgraded with an adaptive stiffness slot.
- `E4`: velocity-dependent drag upgraded from a low-speed linear damping model.
- `E5`: Monod/Michaelis-Menten growth-rate slot with polynomial and rational libraries.
- `E6/E7`: public real-system positive cases for Cascaded Tanks and EMPS friction.
- `E8/E9`: public real-system boundary cases for PMSM thermal and ORNL building data.
- Battery/Silverbox: semi-real engineering and partial-observation boundary diagnostics.
- Ablations: wrong slot, wrong input, function-family mismatch, noise, physical constraints, and complexity/runtime checks.

## Data Policy

Bundled:

- generated synthetic trajectories;
- small processed tables;
- provenance JSON files;
- frozen metrics and summary CSV files;
- experiment figures in PDF/SVG form.

Not bundled:

- large third-party raw datasets;
- temporary QA folders;
- Python bytecode caches;
- high-resolution TIFF/PNG exports that can be regenerated.
